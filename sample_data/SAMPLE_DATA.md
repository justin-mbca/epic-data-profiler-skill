# Sample Data Documentation

This document describes the sample data files included in this project, how they were generated, and their intended use.

## Overview

The `sample_data/` directory contains synthetic Epic-style data files in three formats:
- `sample_epic.csv`
- `sample_epic.json`
- `sample_epic.parquet`

These files are used for testing and demonstrating the Epic data profiling skill.

## How the Sample Data Was Generated

All sample data files are generated by running the script:

```
python sample_data/create_samples.py
```

This script creates 1,000 rows of synthetic patient encounter data with the following columns:
- `patient_id`: Unique integer identifier for each patient
- `encounter_id`: Unique integer identifier for each encounter
- `admit_date`: Random admission date
- `discharge_date`: Random discharge date (after admit_date)
- `age`: Patient age (randomized)
- `gender`: Patient gender (randomized)
- `diagnosis`: Random diagnosis code (randomized)
- `length_of_stay`: Calculated as days between admit and discharge

The script uses Python libraries (`pandas`, `numpy`, `pyarrow`) to generate the data and save it in CSV, JSON, and Parquet formats. All files contain the same data, just in different formats.

## Intended Use

These files are intended for:
- Testing the Epic data profiling skill
- Demonstrating schema inference, type checking, and missing value handling
- Validating the tool-call interface and test suite

## Regenerating the Data

If you need to regenerate the sample data, simply run:

```
python sample_data/create_samples.py
```

This will overwrite the existing files with fresh synthetic data.

## File Descriptions

- `sample_epic.csv`: Comma-separated values, human-readable, can be opened in Excel or any text editor.
- `sample_epic.json`: JSON lines format, each line is a JSON object representing a row.
- `sample_epic.parquet`: Parquet columnar storage, efficient for large data and used in data engineering pipelines.

---

For more details, see the script at `sample_data/create_samples.py`.
